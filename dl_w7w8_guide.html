7.1 · Foundations

Latent Space — The Hidden Code of Reality

Every piece of data in the world is extremely high-dimensional. A 256×256 image has 196,608 numbers. But most of that information is redundant. A latent space is a compressed, meaningful representation that captures the essence of the data in far fewer numbers.

🧬
Analogy — DNA as a Latent Space

Your entire body has trillions of cells, but it's all encoded in ~3 billion DNA base pairs. DNA is the latent space of a human — a compressed code that can reconstruct the entire organism. Similarly, a 512-dimensional latent vector z encodes the "essence" of a 256×256 image. Change one number in z slightly = change the lighting. Change another = change the expression.

Why High-Dimensional Data is Problematic

❌ Raw Pixel Space

  • 256×256×3 = 196,608 dimensions
  • Mostly redundant (wall = all 0s)
  • Computationally insane
  • Noise mixed with signal

✅ Latent Space z

  • Say 128-dimensional vector
  • Each dimension is meaningful
  • z₁ ≈ pose, z₂ ≈ lighting, z₃ ≈ age
  • z ~ N(0, I) → sample → new data!

The "Holes" Problem with Old Methods

Classical methods (PCA, Factor Analysis, etc.) create a latent space for understanding data, but the latent space has holes — large empty regions. If you randomly sample a point from such a space and try to decode it, you'll get garbage output.

The Data Manifold — Why We Can't Just Sample Pixels

Of all possible 256×256 images (256^196608 possibilities), the ones that look like real photos occupy a tiny curved surface — a manifold — in that enormous space. Generative models learn to map a simple distribution (Gaussian noise z) onto this manifold.

Pixel space for 256×256×3 = 196,608 dimensions Possible configurations = 256^196,608 ≈ insanely large number Of all these... "cat images" occupy a tiny sub-manifold. Random pixels → 99.9999% nonsense. Deep generative model solution: 1. Sample from simple distribution: z ~ N(0, I) [say 128-D] 2. Apply learned generator G: x = G(z) [outputs 196,608-D image] 3. G maps the Gaussian blob → the data manifold! → Every z gives a valid (approximately) realistic image

A random z sampled from anywhere may land in a hole, which the decoder has never seen. Modern methods (VAE, GAN) enforce that the entire latent space is densely covered, usually as a Gaussian blob.
7.1 · Classical Methods

PCA, Factor Analysis, LDA & SVD

Before deep learning, people used elegant mathematical tools to discover latent structure in data. These are still used and appear in ML theory frequently.

PCA — Principal Component Analysis

📐
Analogy — Shadow on a Wall

Hold a 3D object in front of a light. Its 2D shadow on the wall captures most of the shape with fewer dimensions. PCA finds the best "angle" to project data so the shadow (lower-dimensional version) captures maximum variance. The principal components are like the axes of the best shadow.

PCA: Find directions (principal components PC1, PC2, ...) that capture maximum variance. Toy example: 2D data (height, weight) of people. Most variance is along the diagonal (tall people tend to weigh more). PC1 ≈ "overall body size" [captures most variance] PC2 ≈ "height/weight ratio" [captures remaining variance] Projecting onto PC1 only → 1D latent space! Applications: Dimensionality reduction, noise removal, visualization, "Eigenfaces" (face recognition)

Factor Analysis — Hidden Traits Behind Test Scores

Model: x = Λf + ε x = observed variables (e.g., test scores) f = latent factors (e.g., "math ability", "memory") Λ = factor loadings (how much each factor influences each test) ε = noise/unexplained part Numerical Example (2 factors, 3 tests): Λ = [[0.9, 0.1], ← Algebra (mostly math) [0.8, 0.2], ← Word problems (math + some memory) [0.2, 0.9]] ← Memory test (mostly memory) For student with f₁(math)=7, f₂(memory)=5: x₁ = 0.9×7 + 0.1×5 = 6.3 + 0.5 = 6.8 (Algebra score) x₂ = 0.8×7 + 0.2×5 = 5.6 + 1.0 = 6.6 (Word problems) x₃ = 0.2×7 + 0.9×5 = 1.4 + 4.5 = 5.9 (Memory test) Insight: Algebra & Word Problems high → because math factor is strong!

LDA — Latent Dirichlet Allocation (Topic Modeling)

Warning: This is not Linear Discriminant Analysis! This LDA is for text — it discovers hidden topics in a collection of documents.

📰
Analogy — Mixing Paint

Every news article is like a paint mixture: 70% Sports blue + 30% Politics red. LDA automatically finds what colors (topics) exist, and how much of each color each article contains — without being told what the topics are beforehand!

LDA key ideas: - Topic = probability distribution over words - Document = mixture of topics Topic 1 (Sports): P("match")=0.2, P("player")=0.3, P("score")=0.25, ... Topic 2 (Politics): P("election")=0.25, P("vote")=0.2, P("party")=0.15, ... Document "The player scored in the final match": θ_doc = [0.9 Sports, 0.1 Politics] Latent representation of the document = θ_doc = [0.9, 0.1] (This is the latent vector!)

🔢 Matrix Factorization — Recommend Movies

SVD (Singular Value Decomposition): A = U · Σ · Vᵀ U = left singular vectors (m×m orthogonal) Σ = diagonal matrix of singular values σ₁ ≥ σ₂ ≥ ... ≥ 0 Vᵀ = right singular vectors (n×n orthogonal) Low-rank approximation: Keep only top-K singular values: A ≈ Σₖ σₖ uₖ vₖᵀ (sum of K rank-1 matrices) Example: A = [[3,0],[0,1]] Σ = diag(3,1), U = I, V = I Keep only top-1 singular value: A ≈ [[3,0],[0,0]] → Second dimension ignored → dimensionality reduction!
🔗 Common Theme in All Classical Methods
  • Factor Analysis: x = Λf + ε — observed = weighted sum of hidden factors
  • LDA: document = mixture of topics; topic = distribution over words
  • Matrix Factorization / SVD: Big R ≈ U·Vᵀ — user/item latent vectors
  • All share: "High-dimensional observations ≈ few hidden dimensions explain everything"

AE Loss = Reconstruction error: L_recon = ‖x − x̂‖² Numerical Example: x = [1, 0, 0], x̂ = [0.8, 0.1, 0.0] L = (1-0.8)² + (0-0.1)² + (0-0.0)² = 0.04 + 0.01 + 0 = 0.05 Problem: Latent space has NO distribution constraint! VAE — Adding a Gaussian Constraint on the Latent Space

VAE fixes AE's problem by forcing the encoder to output a distribution, not a single point, and regularizing that distribution to be close to a standard Gaussian.

VAE Loss = Reconstruction Loss + KL Divergence penalty: L = E[‖x − x̂‖²] ← "remember the original data" + D_KL(q(z|x) ‖ p(z)) ← "keep latent space close to N(0,I)" KL Divergence intuition (1D case): p(z) = N(0,1) [target/prior] q(z|x) = N(μ, σ²) [encoder output] If μ=0, σ=1: KL = 0 ✅ (perfect match to prior) If μ=2, σ=1: KL > 0 ❌ (shifted away, penalty applied) If μ=0, σ=0.1: KL > 0 ❌ (too narrow, penalty applied) VAE training forces the encoder to stay near N(0,I) in latent space → After training, any z ~ N(0,I) gives a valid-ish reconstruction! Reparameterization trick: z = μ(x) + σ(x) ⊙ ε, ε ~ N(0,I) → Allows gradients to flow back through the sampling operation!
💡 VAE = AE + Probabilistic Latent Space. AE learns a deterministic point z per input. VAE learns a distribution q(z|x) per input, regularized to be ~N(0,1). This fills in the "holes" and makes random sampling meaningful.
7.4 · Core Model

GAN — The Adversarial Game

🎭
Analogy — Counterfeiter vs. Detective

Generator G = Counterfeiter: tries to print fake currency that looks real.
Discriminator D = Detective: tries to spot the fake bills.

As the detective gets better at spotting fakes, the counterfeiter must improve. As the counterfeiter improves, the detective must sharpen their skills further. This competition drives both to excellence — until the counterfeiter's fakes are indistinguishable from real currency!

The Math: Minimax Objective

min_G max_D V(D,G) = E[log D(x)] + E[log(1 − D(G(z)))] Discriminator D wants to MAXIMIZE this: → D(real) → 1: log(1) = 0 (good) → D(fake) → 0: log(1-0) = 0 (good) Generator G wants to MINIMIZE this: → Make D(G(z)) → 1: log(1-1) = -∞ (minimize → maximize D's confusion) Practical G objective (avoid vanishing gradient!): Instead of: minimize log(1 - D(G(z))) ← gradient ≈ 0 when D is confident Use: maximize log(D(G(z))) ← strong gradient even when D is sure
🔢 D Loss Numerical Example

Batch: 2 real images, 2 fake images

Image typeD scoreContribution to D loss
Real x₁D(x₁) = 0.9log(0.9) ≈ −0.105
Real x₂D(x₂) = 0.8log(0.8) ≈ −0.223

D loss = −(sum of all log terms). D is doing well here (real→high, fake→low). If D gives real=0.4, fake=0.7, the log terms become very negative → large loss → D forced to improve!

GAN Training Loop (Alternating Optimization)

1
Train Discriminator D (freeze G): Sample real batch x_real from dataset. Generate fake batch x_fake = G(z). Update D to correctly label real→1 and fake→0. L_D = −[log D(x_real) + log(1 − D(x_fake))]
2
Train Generator G (freeze D): Sample new noise z. Generate x_fake = G(z). Update G to make D think fake is real. L_G = −log D(G(z)) [non-saturating version]
3
Ideal end state: G has learned P_g(x) ≈ P_data(x). D can't distinguish real from fake → D(x) = 0.5 everywhere. This is called the Nash Equilibrium.

Normal GAN: z → Generator → image (hard to control what changes) StyleGAN: z → Mapping Network (8 FC layers) → w → style vectors per layer → image Key innovations: 1. Mapping network: z (entangled) → w (disentangled latent) w = more organized: moving w₁ changes ONLY hairstyle, w₂ changes ONLY age, etc. 2. Diffusion Models — Forward Noise + Learn to Reverse

🌫️
Analogy — Watercolor Smearing and Un-smearing

Forward process: Take a crisp painting and add a tiny bit of water. Then again. Then again... 1000 steps later it's just a uniform gray blur (pure Gaussian noise).
Reverse process: A neural network learns to UN-smear — given a slightly blurry image, predict the slightly less blurry version. After 1000 reverse steps, you get a crisp new painting! Stable Diffusion, DALL·E 3, and Midjourney all use this.

Forward (adds noise, step by step): xₜ = √αₜ · xₜ₋₁ + √(1-αₜ) · ε, ε ~ N(0,I) After T steps: xₜ ≈ pure Gaussian noise N(0,I) Reverse (learned denoiser): Network ε_θ(xₜ, t) predicts the noise added at step t Loss: E[‖ε − ε_θ(xₜ, t)‖²] ← simple MSE on noise prediction! Sampling: Start from xₜ ~ N(0,I) → apply T reverse denoising steps → get x₀ Pros: High-quality diverse samples, stable training Cons: T=1000 steps per image → slow sampling
🧠 Quick Check

What is "mode collapse" in GAN training?

A The discriminator collapses to always outputting 0.5
B Training loss goes to zero and then spikes back up
C The generator produces images with all-black pixels
D The generator produces very limited variety — different inputs z give nearly identical outputs
Mode collapse: G finds a small set of outputs that reliably fool D, and keeps generating only those. E.g., training on digits 0-9 but G always outputs "3". G has found a "mode" of the distribution and "collapsed" to it. Mini-batch discrimination forces D to look at a whole batch at once — if they all look the same, D knows they're fake!
📋 Week 7 Complete Summary
  • Latent space: Compressed, meaningful lower-D representation. z ~ N(0,I) → Generator → new data
  • Data manifold: Real images live on a tiny curved surface in pixel space. Generative models map Gaussian noise onto it
  • Classical methods: PCA (max variance axes), Factor Analysis (x=Λf+ε), LDA (doc=topic mix), SVD (A≈UΣVᵀ)
  • Generative taxonomy: Explicit (tractable: Autoregressive, Flows) vs Approximate (VAE, Diffusion) vs Implicit (GAN)
  • AE: x→z→x̂, loss=MSE. No distribution constraint → bad for generation
  • VAE: Encoder outputs μ(x),σ(x). Loss = recon + KL(q(z|x)‖N(0,I)). Structured latent space → random z gives valid output
  • GAN: G (counterfeiter) vs D (detective). min_G max_D V(D,G). Nash eq: D(x)=0.5 everywhere
  • GAN issues: Mode collapse, non-convergence, vanishing gradients. Fix: WGAN, mini-batch discrimination, progressive growing
  • StyleGAN: z→w (mapping network) + AdaIN + per-layer noise = controllable face generation
  • Diffusion: Forward=add noise 1000 steps. Reverse=learned denoiser. DALL·E 3, Stable Diffusion use this!

7.1 & 8.1 · Foundations

Why RNNs? The Problem with Sequences

So far, every model we've built assumed independent data points (IID). But real-world data is often a sequence, where each element depends on what came before it. Enter the Recurrent Neural Network.

🎬
Analogy — Reading a Mystery Novel

You can't understand what "He killed him" means on page 200 without context from earlier in the book. Similarly, a word in a sentence, a frame in a video, or a price on a stock chart only makes sense in the context of what came before. RNNs are designed to maintain this context as they process a sequence, one element at a time.

Sequence Data: 3 Key Characteristics

A
Order matters: "Rama killed Ravan" ≠ "Ravan killed Rama". Same words, totally different meaning. Stock going 100→80→120 (crashed then recovered) ≠ 120→80→100 (peaked then dropped).
B
Variable length: "Nice movie" (2 tokens) vs "The movie was extremely boring and slow" (8 tokens). A Fitbit recording for 1 hour (3600 steps) vs 15 minutes (900 steps). Simple ANNs need fixed-size input — problem!
C
Short & long-range dependencies: "I am very happy" (short: "very"→"happy"). "Gapati and Balaji taught a class on GenAI. They took 12 weeks." (long: "they" refers to something 10+ words back). Both need to be captured.

Why ANNs / CNNs Can't Handle Sequences Well

ProblemANN/CNN IssueRNN Solution
Variable length inputNeed fixed-size vector. Truncation loses info. Padding adds fake zeros.Process one token at a time, any length T!
No time-based parameter sharingDifferent weights for position 1 vs position 2 (implicit)Same weights W used at every time step
No memory of contextSingle feedforward pass, no recurrenceHidden state hₜ = "memory" of everything seen so far

Types of Sequence Tasks

Sequence → Single Output

  • Sentiment analysis (review → ⭐ rating)
  • Time series prediction (past prices → next price)
  • Video classification (frames → "running")
  • Predictive maintenance (sensor → failure prob)

Sequence → Sequence

  • Machine translation (English → Hindi)
  • Named Entity Recognition (word → label per word)
  • Speech recognition (audio → text)
  • Music generation (notes → more notes)
🧠 Quick Check

Why is zero-padding a poor solution for handling variable-length sequences in a standard ANN?

A Zeros cause division by zero errors in backpropagation
B Padding makes the input too small
C Padded zeros are meaningless but still influence weights via multiplication and gradients, confusing training
D Padding only works for images, not text
Padding zeros are fake data. These zeros still get multiplied by weight matrices, contribute to activations, and influence gradients during backpropagation. The model has to somehow "learn" that zeros at certain positions are meaningless — which is very hard! RNNs avoid this entirely by processing sequences step by step, stopping naturally when the sequence ends.
8.2 · Language Models

Language Models: Teaching AI to Understand Text

A language model is a sequence model applied to text. Its core question: How likely is this sequence of words? This probability estimation is literally the engine inside ChatGPT.

Step 1: Tokenization — Breaking Text into Pieces

Character-Level

"split" → [s, p, l, i, t] = 5 tokens

Vocabulary ≈ 50 chars
Sequences are very long
Works for any language/code

Word-Level ⭐

"split this text" → ["split", "this", "text"] = 3 tokens

Vocabulary ≈ 300,000 words
Shorter, more meaningful sequences
Can't handle unknown words well

Step 2: Joint Probability via Chain Rule

Language Model: estimate P(w₁, w₂, ..., wₜ) for any sequence. Chain Rule (always exact): P(w₁, w₂, ..., wₜ) = P(w₁) · P(w₂|w₁) · P(w₃|w₁,w₂) · ... · P(wₜ|w₁,...,wₜ₋₁) Real-world analogy (weather chain rule): P(rain, umbrella, wet) = P(rain) × P(umbrella|rain) × P(wet|rain,umbrella) = 0.3 × 0.8 × 0.2 = 0.048 Language example: "the cat sat" P("the") = 0.2 P("cat"|"the") = 0.5 P("sat"|"the cat") = 0.8 P("the cat sat") = 0.2 × 0.5 × 0.8 = 0.08 Compare: "the cat sang" P("sang"|"the cat") = 0.01 P("the cat sang") = 0.2 × 0.5 × 0.01 = 0.001 Higher probability → more natural sentence! (0.08 >> 0.001)

Step 3: Bigram Model — A Practical Approximation

The full chain rule requires conditioning on all previous words — too expensive! N-gram models approximate by only looking at the last n-1 words:

Bigram assumption (n=2): P(wₜ | w₁,...,wₜ₋₁) ≈ P(wₜ | wₜ₋₁) Estimate from counts: P(wₜ | wₜ₋₁) = count("wₜ₋₁ wₜ") / count("wₜ₋₁ *") Example corpus: "the cat sat" + "the cat slept" + "the dog sat" Bigrams and counts: ("the","cat") → 2, ("the","dog") → 1, ("cat","sat") → 1, ... P("cat"|"the") = 2/3 ≈ 0.67 P("dog"|"the") = 1/3 ≈ 0.33 P("sat"|"cat") = 1/2 = 0.50 P("slept"|"cat") = 1/2 = 0.50 Probability of "the cat sat": P = P("the") × P("cat"|"the") × P("sat"|"cat") = 1 × 0.67 × 0.50 = 0.33

Step 4: Creating Training Data (X, Y Pairs)

Language model training is self-supervised — no human labels needed. We create (X, Y) pairs from the raw text using next token prediction:

🔢 Creating X, Y Subsequences with Window Size τ=3

Corpus: "sat on the mat and slept all day" → tokens: [3, 4, 1, 5, 6, 7, 8, 9]

Pair #X (input tokens)Y (target = X shifted by 1)Meaning

8.3 · Evaluation

Perplexity — How "Confused" Is the Language Model?

Perplexity (PPL) tells us how well a language model predicts text. It's the standard metric — lower is always better.

🎲
Analogy — Loaded Dice

Perplexity ≈ effective number of equally likely choices the model faces at each step. If PPL = 100, the model behaves as if it has 100 equally likely options for the next word. A perfect model (knows exactly what comes next) has PPL = 1. A model that randomly guesses from a vocabulary of V words has PPL = V.

8.4 · Architecture

RNN Architecture: The Hidden State as Memory

An RNN processes a sequence one element at a time, maintaining a hidden state hₜ that acts as the "memory" of everything it has seen so far.

🌊
Analogy — Straw in a River

Imagine a river of information (x₁, x₂, x₃, ...) flowing past. At each point, the RNN updates its hidden state hₜ. You can put a "straw" anywhere in the river to extract an output — at the end (sentiment), at every step (next word prediction), or at any specific step. The hidden state is like a continuously updated summary of the river so far.

The Core Equations

At each time step t, RNN does: (1) Compute hidden state: zₜ = W_xh · xₜ + W_hh · hₜ₋₁ + b_h hₜ = tanh(zₜ) ← or ReLU (2) Compute output (optional, depends on task): oₜ = W_hy · hₜ + b_y yₜ = softmax(oₜ) ← next word prediction yₜ = sigmoid(oₜ) ← binary classification yₜ = oₜ ← regression Three weight matrices (learned, SAME at every time step!): W_xh: (hidden_size × input_size) ← "current input encoding" W_hh: (hidden_size × hidden_size) ← "memory update" W_hy: (output_size × hidden_size) ← "judgment/decision" Initialize: h₀ = zeros (or learnable start token)

The Three Weight Matrices — Dimensions

🔢 Concrete Dimension Example

Input: 5-D weather vector (temp, pressure, humidity, wind, cloud). Hidden size: 100. Task: binary prediction (rain tomorrow?)

W_xh100 × 5500"Encode today's weather"
W_hh100 × 10010,000"Update memory with history"

Total trainable parameters = 10,701. And ALL of these are reused at EVERY time step — that's parameter sharing! For next-word prediction over vocab V=10,000: W_hy would be 10,000 × 100 = 1,000,000 params.

Full Numerical Walkthrough

Tiny RNN: input_dim=2, hidden_dim=3, h₀=[0,0,0] W_xh = [[1,0],[0,1],[1,-1]] W_hh = [[0.5,0,0],[0,0.5,0],[0,0,0.5]] b_h = [0,0,0] Sequence: x₁=[1,2], x₂=[0,1] ── Step 1 (t=1) ── z₁ = W_xh·x₁ + W_hh·h₀ + b_h W_xh·x₁ = [1×1+0×2, 0×1+1×2, 1×1+(-1)×2] = [1, 2, -1] W_hh·h₀ = [0, 0, 0] (h₀=zeros) z₁ = [1, 2, -1] + [0,0,0] = [1, 2, -1] h₁ = tanh([1, 2, -1]) ≈ [0.76, 0.96, -0.76] ── Step 2 (t=2) ── W_xh·x₂ = [1×0+0×1, 0×0+1×1, 1×0+(-1)×1] = [0, 1, -1] W_hh·h₁ ≈ [0.5×0.76, 0.5×0.96, 0.5×(-0.76)] = [0.38, 0.48, -0.38] z₂ = [0,1,-1] + [0.38,0.48,-0.38] = [0.38, 1.48, -1.38] h₂ = tanh([0.38, 1.48, -1.38]) ≈ [0.36, 0.90, -0.88] h₂ now contains compressed info from BOTH x₁ AND x₂! 🎉
💡 Why "Recurrent"? Because hₜ feeds back into the computation of hₜ₊₁. The hidden state refers to (recurs on) itself. It's a self-referential, auto-regressive process: each hidden state is a function of the previous hidden state. This is what gives RNNs their "memory."

RNN vs n-gram: The Hidden State Trick

n-gram model: P(xₜ | xₜ₋₁, ..., xₜ₋ₙ₊₁) ✗ Only last n-1 tokens visible (short memory) ✗ Table grows exponentially: V^n entries needed ✗ Trigram with V=50k: 125 billion entries! RNN model: P(xₜ | hₜ₋₁) ✓ h_{t-1} = compressed "zip file" of ENTIRE past history ✓ Fixed-size vector regardless of sequence length ✓ No huge table — learned parameters instead ✓ Can (in theory) capture arbitrarily long dependencies "The hidden state h_{t-1} is a single vector representing the entire sequence x₁, ..., xₜ₋₁." — Lecture

In an RNN with input_dim=10, hidden_dim=50, output_dim=20, how many parameters does W_hh have?

A 500 (50 × 10)

W_hh maps hidden_dim → hidden_dim, so its shape is 50 × 50 = 2500 parameters. (W_xh = 50×10 = 500; W_hy = 20×50 = 1000). The hidden-to-hidden matrix is almost always the largest — it captures how the memory evolves over time.

📋 Week 8 Complete Summary
  • Sequence data: Order matters. IID assumption breaks. Variable length. Short & long range dependencies.
  • Why RNN: ANN can't handle variable length, no temporal parameter sharing, no memory. RNN solves all three.
  • Tokenization: Text → tokens → integer IDs. Char-level (small vocab, long seqs) vs Word-level (large vocab, short seqs).
  • Vocabulary: Unique tokens. UNK=0 for unknown words. Corpus size ≠ vocabulary size.
  • Language model: P(w₁,...,wₜ) via chain rule = P(w₁)·P(w₂|w₁)·... Higher P = more natural sentence.
  • Bigram: P(wₜ|wₜ₋₁) ≈ count(wₜ₋₁,wₜ)/count(wₜ₋₁,*). Simple but limited context.
  • Training data: X=[x₁,...,xτ], Y=[x₂,...,xτ₊₁] (shifted by 1). Self-supervised = no human labels!
  • Perplexity: PPL = P(seq)^(−1/T). Lower = better. PPL=1 perfect. PPL=V random. PPL≈k means k-way confusion.
  • RNN equations: zₜ = W_xh·xₜ + W_hh·hₜ₋₁ + bₕ; hₜ = tanh(zₜ); yₜ = g(W_hy·hₜ + bᵧ)
  • Parameter sharing: Same W_xh, W_hh, W_hy at EVERY time step → works for any sequence length!
  • RNN trick: P(xₜ|x₁,...,xₜ₋₁) ≈ P(xₜ|hₜ₋₁). Hidden state = compressed "zip file" of full history.
🎯 Final Mixed Quiz — Weeks 7 & 8

1. What is the key difference between a VAE encoder and a standard AE encoder?

A VAE encoder is much larger than AE encoder
B AE encoder uses sigmoid activation; VAE uses ReLU
C VAE encoder outputs distribution parameters μ and σ, not a single point z
D VAE encoder works without backpropagation
AE encoder: x → z (deterministic single point). VAE encoder: x → μ(x), σ(x) (parameters of a Gaussian distribution). Then z is sampled: z = μ + σ⊙ε, ε~N(0,I). This probabilistic encoding, combined with KL regularization, gives VAE its structured latent space.

2. In GAN training with the original G loss, why does the generator struggle when D is very good early in training?

A D(G(z))≈0 → log(1−D(G(z)))≈log(1)=0 → gradient≈0 → G can't learn (vanishing gradient)
B The generator's learning rate is too high
C Mode collapse makes all outputs identical from the start
D The discriminator is always frozen when training the generator
When D is confident fakes are fake, D(G(z))≈0. The original G loss is log(1-D(G(z))) = log(1-0) = log(1) = 0. The gradient of log(1-y) at y≈0 is ≈0 — so G receives almost no learning signal! The fix: use maximize log D(G(z)) instead. At y=0.01: log(0.01) = -4.6, gradient is large and G learns fast.

3. What is the "reparameterization trick" in VAE and why is it needed?

A It converts the encoder from CNN to MLP
B It changes the loss function from MSE to KL divergence
C It allows the model to use smaller batch sizes
D z=μ+σ·ε (ε~N(0,I)) — moves the randomness to a fixed input, allowing gradients to flow through the sampling operation
You can't backpropagate through a random sampling operation (sampling z~N(μ,σ²) is not differentiable). The trick: write z=μ+σ·ε where ε~N(0,I) is sampled independently. Now z is a deterministic function of μ and σ (which the network computes), and ε is just a fixed external noise input. Gradients flow through μ and σ as usual!

4. Why do RNNs use the SAME weight matrices W_xh, W_hh, W_hy at every time step?

A Using different weights per step would make the network too accurate
B The relationship between adjacent elements is assumed to be the same regardless of position — enabling generalization to any sequence length
C GPUs can only process one weight matrix at a time
D Shared weights prevent overfitting only in long sequences
Parameter sharing assumes stationarity — the same "next word given previous word" mechanism applies at every position. This enables: (1) training on sequences of any length with the same parameters, (2) fewer parameters = less overfitting, (3) the model can generalize to sequences longer than those seen during training. Same idea as CNN sharing the same kernel across all spatial positions.
