5.1 · Foundations

The Deep Learning Pipeline

Every single deep learning model — whether it classifies images, generates text, or recommends videos — follows the same core loop. Understanding this loop is the foundation of everything else.

🎯
Analogy — Archery Practice

Imagine you're learning archery. You shoot an arrow (prediction), see where it lands vs where the bullseye is (loss), and then adjust your stance/grip (update weights). Repeat until you consistently hit the target. That's deep learning training in a nutshell!

The Gradient Descent Update Rule

This is the single most important formula in all of deep learning. Memorize it:

Wnew = Wold − η · ∇W L(Wold) Where: η (eta) = Learning rate → "how big a step to take" ∇_W L = Gradient → "which direction is uphill?" Minus sign = We go downhill (minimize loss)
🔢 Solved Example — 2 Steps of Gradient Descent

Loss function: L(w) = (w − 3)²   |  Gradient: dL/dw = 2(w − 3)   |  Start: w=0, η=0.1

Stepw_oldGradient = 2(w−3)Update = w − 0.1 × gradw_new
102(0−3) = −60 − 0.1×(−6) = 0 + 0.60.6
20.62(0.6−3) = −4.80.6 − 0.1×(−4.8) = 0.6 + 0.481.08
31.082(1.08−3) = −3.841.08 − 0.1×(−3.84)1.464
✅ Each step, w gets closer to the true minimum at w = 3. The gradient points us in the right direction!

Training Error vs. Generalization Error — The Real Goal

Empirical Risk (Training Error)

What we actually measure. Average loss over our training dataset. The optimizer minimizes this.


R_emp = (1/n) Σ L(f_W(xᵢ), yᵢ)

True Risk (Generalization Error)

The real goal — how well does the model work on unseen data? We can never measure this directly.


R_true = E[L(f_W(x), y)]
🗳️
Analogy — Exit Poll vs Actual Election

Exit poll (survey of 2000 people) = Empirical risk (your training data). Actual election result (all voters) = True risk (real world). Just because your training loss is zero doesn't mean you'll perform well on new data — just like exit polls can be wrong!

⚠️ Key Insight: The minimum of training loss and the minimum of true risk can be at different weight values. This is why overfitting happens — you minimized training error but not generalization error. Always use a validation set!
🧠 Quick Check

The gradient ∇L points in the direction of steepest increase. Since we want to MINIMIZE the loss, we move in the opposite direction — hence the minus sign!
5.1 · Non-Convex Challenges

3 Big Problems with Optimization

Real neural networks have non-convex loss landscapes — imagine a crazy mountain range with fog everywhere. Here are the three monsters you face:

Problem 1: Local Minima

A local minimum is a point that looks like the bottom of a valley nearby, but there's a deeper valley somewhere else (the global minimum).

f(x) = x⁴ − 4x² + 3 Evaluate at key points: f(0) = 3 ← local maximum f(1) = 0 ← local minimum f(-1) = 0 ← local minimum (Both x=±1 happen to be global minima for this function) Point: gradient descent starting at x=4 vs x=-4 may find different local minima — starting point matters!

Problem 2: Saddle Points

At a saddle point, the gradient = 0 (so GD stops!), but it's not a minimum. It's a minimum in one direction but a maximum in another — like the middle of a horse saddle.

🔢 Saddle Point Example: f(x,y) = x² − y²

At the point (0, 0):

DirectionFunction behavior(0,0) looks like...
Along x-axis (y=0)f = x² → curves upwardMinimum 🟢
Along y-axis (x=0)f = -y² → curves downwardMaximum 🔴
∂f/∂x = 2x = 0 and ∂f/∂y = -2y = 0 at (0,0) → gradient is zero, but it's a SADDLE, not a minimum!

Problem 3: Vanishing Gradients (Most Dangerous!) 💀

In deep networks using Sigmoid or tanh activations, the gradient gets multiplied by a small number at each layer during backpropagation. After many layers, it becomes so tiny the machine treats it as zero — and learning stops.

📠
Analogy — Photocopy of a Photocopy

Imagine photocopying a document, then photocopying the photocopy, then photocopying that... After 10 rounds, the text is unreadable. That's what happens to gradients when they flow backward through 10 sigmoid layers — each multiplication by 0.25 makes them fainter and fainter!

🔢 Sigmoid Vanishing Gradient — Numerical Proof

Sigmoid derivative max = 0.25. With 10 layers, the gradient at layer 1 is:

Gradient shrinkage per layer: 0.25 (best case!) (0.25)^10 = ? 0.25^2 = 0.0625 0.25^4 = 0.00390625 0.25^8 ≈ 0.0000153 0.25^10 ≈ 0.000000953 ≈ 10⁻⁶ If final layer gradient = 10⁻³ (reasonable): Layer 1 gradient ≈ 10⁻³ × 10⁻⁶ = 10⁻⁹ Float32 machine precision ≈ 10⁻⁷ → 10⁻⁹ appears as ZERO! Learning has stopped for early layers. 😱

The Solution: ReLU Activation

ReLU(z) = max(0, z) Derivative: ReLU'(z) = 1 (when z > 0) ← gradient passes through unchanged! ReLU'(z) = 0 (when z ≤ 0) ← neuron is "off" Compare to Sigmoid's max derivative of 0.25: Sigmoid: 0.25 × 0.25 × 0.25 = 0.015625 after 3 layers ReLU: 1 × 1 × 1 = 1.000000 after 3 layers ✅

ResNet Skip Connections — Another Solution

ResNets add a "highway" for the gradient — it can skip layers and travel directly:

Normal: output = F(x) ← gradient must flow through F ResNet: output = F(x) + x ← gradient has TWO paths! During backprop: Path 1: Through F(x) (may be small) Path 2: Through "+x" identity shortcut → gradient = 1 (never vanishes!) ✅ This is why 100+ layer networks became possible after ResNet (2015).
🧠 Quick Check

Why does sigmoid activation cause vanishing gradients in deep networks?

A Because sigmoid outputs are always negative
B Because sigmoid gradients can exceed 1 and cause overflow
C Its maximum derivative is 0.25, so repeated multiplication makes gradients exponentially small
D Sigmoid is only defined for positive inputs
Correct! Sigmoid's max derivative is 0.25. With L layers, gradient ≈ (0.25)^L. For L=10, that's ~10⁻⁶, which becomes numerically zero on most hardware. ReLU's derivative of 1 (for active neurons) solves this!
5.2 · Theory

Convex Optimization

Even though deep learning is non-convex, we study convex optimization because: (1) it has beautiful theory with guaranteed answers, (2) simple cases like linear regression are convex, and (3) locally, non-convex functions often look convex. It's our playground for testing ideas.

What is a Convex Set?

🫙
Analogy — The Rubber Band Test

Pick any two points inside a shape and draw a straight line between them. If the entire line stays inside the shape, it's convex. A filled circle: ✅ convex. A crescent/C-shape: ❌ not convex (the line would exit the shape).

Formal definition: A set X is CONVEX if for any two points A, B ∈ X and any λ ∈ [0, 1]: λA + (1-λ)B also belongs to X Think of it: λ goes from 0→1, tracing the line from B to A. The condition says: every point on that line must be inside X. Example: The square [0,1] × [0,1] A=(0,0), B=(1,1), λ=0.5: midpoint = (0.5, 0.5) ✅ inside square ✅ Square is convex!

What is a Convex Function?

A function is convex if its graph always lies below or on the chord connecting any two points. Imagine a "smile" shape — like a bowl (U-shape).

f is CONVEX if for all x, x' and λ ∈ [0,1]: f(λx + (1-λ)x') ≤ λf(x) + (1-λ)f(x') ↑ function of midpoint ↑ midpoint of function values Geometric intuition: The curve is always BELOW the chord. Check for f(x) = x² at x=0, x'=2: Left side: f(0+2)/2) = f(1) = 1 Right side: (f(0)+f(2))/2 = (0+4)/2 = 2 Is 1 ≤ 2? YES ✅ → x² is convex!

✅ Convex Functions

  • f(x) = x² (parabola)
  • f(x) = x⁴ (always U-shaped)
  • f(x) = eˣ (exponential)
  • f(x) = |x| (absolute value)

❌ Non-Convex Functions

  • f(x) = cos(πx) (wavy)
  • f(x) = x³ (S-curve)
  • Loss landscape of deep nets
  • sin(x) (oscillates)

🌟 The Golden Property of Convex Functions

Every local minimum of a convex function is also the global minimum! This means gradient descent on a convex function is guaranteed to find the best solution. No nasty surprises!

Jensen's Inequality

A powerful result: for a convex function f, the average of the function values is always ≥ the function of the average.

🔢 Jensen's Inequality — Numerical Example

Let X take values {1, 3} with probability 0.5 each. Let f(x) = x² (convex).

QuantityCalculationResult
E[X] (average of X)(1+3)/22
f(E[X])f(2) = 2² = 44
E[f(X)] (average of f values)0.5×f(1) + 0.5×f(3) = 0.5×1 + 0.5×95
E[f(X)] = 5 ≥ f(E[X]) = 4 ✅ Jensen's inequality: E[f(X)] ≥ f(E[X]) for convex f

Convexity Check: Second Derivative Test

1D: f(x) is convex if f''(x) ≥ 0 for all x (Second derivative ≥ 0 means "always curving upward") Example: f(x) = x² → f'(x)=2x → f''(x)=2 ≥ 0 ✅ convex Example: f(x) = x³ → f'(x)=3x² → f''(x)=6x → negative for x<0 ❌ not convex Multi-D: f(x) is convex if the Hessian matrix H is positive semidefinite (All eigenvalues of H ≥ 0) Example: f(x₁,x₂) = x₁² + 2x₂² Hessian H = [[2, 0], [0, 4]] Eigenvalues = 2 and 4 (both > 0) → Positive definite → CONVEX ✅

Constrained Optimization (Quick Overview)

Sometimes weights must satisfy constraints (e.g., w ≥ 0). Three approaches:

1
Lagrangian: Add constraint as a penalty term with multiplier λ. Used heavily in SVM theory: L(x,λ) = f(x) + λ·g(x)
2
Penalty / Regularization: Add ‖w‖² to the loss (L2 regularization). Big weights get penalized — keeps the model simple. This is what weight decay is!
3
Projection: Take a GD step, then "snap" the result back to the allowed region. E.g., if constraint is w ∈ [0,1]: w = clip(w_new, 0, 1)
5.3 · Core Algorithm

Gradient Descent — Deep Dive

We saw the basic formula. Now let's understand the geometry, multi-dimensional case, and its variants.

Why Negative Gradient = Fastest Descent (Math Proof!)

🌡️
Analogy — Finding the Cold Spot in a Room

You're blindfolded in a room with a heater and AC. You want to find the coldest spot (AC). At every position you can feel the temperature gradient (which direction is warmer). You move in the opposite direction of the gradient — toward cold! That's gradient descent.

Using the Taylor expansion: f(x + Δx) ≈ f(x) + ∇f · Δx

Change in f ≈ ∇f · Δx = ‖∇f‖ · ‖Δx‖ · cos(θ) Where θ = angle between step direction and gradient. To MINIMIZE the change in f, we need cos(θ) as negative as possible. cos(θ) is most negative when θ = 180° → step is exactly OPPOSITE to gradient! Therefore: Δx ∝ −∇f → Step in negative gradient direction This gives maximum decrease in f — "steepest descent".

Learning Rate: The Goldilocks Problem

🔢 Effect of Learning Rate on f(x) = x²

Starting from x₀=10. Update: x_new = x_old × (1 − 2α)

α (learning rate)Factor (1−2α)PathResult
0.05 (too small)0.910 → 9 → 8.1 → ...Slow convergence
0.2 (just right)0.610 → 6 → 3.6 → 2.16 → ...Fast, stable ✅
1.1 (too big)-1.210 → -12 → 14.4 → -17.28 ...Diverges! 💥
Sweet spot: big enough to converge fast, small enough not to overshoot the minimum.

Multi-Dimensional GD: The Ellipse Problem

Real neural networks have millions of parameters. When the loss landscape is more curved in some directions than others (called bad conditioning), GD zig-zags inefficiently.

🔢 Bad Conditioning Example: f(x₁,x₂) = x₁² + 2x₂²

Start: (x₁,x₂)=(4,4), α=0.1. Gradients: [2x₁, 4x₂]

ParameterGradientUpdate (−0.1 × grad)New value
x₁2×4 = 8−0.83.2
x₂4×4 = 16−1.62.4
x₂ moves TWICE as fast as x₁ due to higher curvature. With extreme conditioning (like x₁² + 20000x₂²), the same learning rate causes x₂ to explode while x₁ barely moves. This motivates adaptive learning rates!

Newton's Method — Using Curvature (2nd Order)

Gradient Descent (1st order): x_new = x − η · ∇f(x) Newton's Method (2nd order): x_new = x − H⁻¹ · ∇f(x) H = Hessian matrix (matrix of all second derivatives) H⁻¹ = inverse of Hessian Why is Newton's method faster? It uses curvature information to take the PERFECT step size! Example: f(x) = x² + 2 f'(x) = 2x, f''(x) = 2 Newton step: x_new = x - 2x/2 = x - x = 0 Starting from x=10: ONE step to reach minimum! 🚀 But in deep learning with n=10⁶ parameters: H is 10⁶ × 10⁶ = 10¹² entries → IMPOSSIBLE to store! So we mostly use first-order methods (GD variants) instead.

5.4 · Hyperparameters

Learning Rate Scheduling

A constant learning rate is rarely optimal. The trick is to start large (fast movement toward minimum) and shrink over time (precise fine-tuning).

🛬
Analogy — Landing an Airplane

When far from the airport, the plane descends quickly. As it approaches the runway, it slows down dramatically for a precise, smooth landing. Same idea with learning rates — start fast, slow down as you approach the minimum!

Types of Learning Rate Schedules

Schedule TypeFormulaBehaviorBest For
Piecewise Constantη = η₀ until step T₁, then η₁...Drops in stepsSimple, intuitive schedules
Exponential Decayη_t = η₀ · γᵗ (e.g. γ=0.9)Smooth exponential dropMost common, easy to tune
Cosine Annealingη_t = η_min + ½(η_max−η_min)(1+cos(πt/T))Smooth curve, can cycleState-of-the-art models
Warmup + DecaySmall LR → Big LR → DecayRise then fallTransformers, large models

Each step, LR shrinks by 10%. This prevents overshooting the minimum as training progresses.
🌀 Cosine Annealing Intuition: Sometimes the scheduler increases the LR slightly during training. This helps the optimizer escape local minima by jumping a bit — like warming metal to find a better crystal structure (annealing). Then it cools again for precision.
5.4 · Optimization

Momentum — Learning from History

Normal GD only looks at the current gradient. Momentum keeps a "running average" of past gradients, like a ball gaining speed rolling downhill.

⚽
Analogy — Rolling Ball on a Hill

A ball rolling down a hill doesn't just respond to the current slope — it has momentum (inertia). If it was going fast in one direction, it keeps going that way unless the slope is very different. Small bumps don't throw it off course. This is exactly what momentum does for gradient descent!

Momentum Equations

Normal GD: w_new = w_old − η · ∇J(w) Momentum GD (adds velocity v): v_t = β · v_{t-1} + ∇J(w_t) ← update velocity w_{t+1} = w_t − η · v_t ← update weights β = momentum coefficient (typically 0.9) β = 0 → same as normal GD (no memory) β → 1 → lots of memory (past gradients matter a lot)

Unrolling the velocity equation reveals that v_t is an exponentially weighted average of past gradients — recent ones count more, old ones fade away:

v_3 = g_3 + β·g_2 + β²·g_1 + β³·g_0 With β=0.9: g_3 gets weight 1 (current gradient, full weight) g_2 gets weight 0.9 (1 step old) g_1 gets weight 0.81 (2 steps old) g_0 gets weight 0.729 (3 steps old → nearly forgotten)
🔢 Momentum vs Normal GD on f(w)=w², w₀=10, η=0.2, β=0.9
StepNormal GDMomentum GDExplanation
Startw=10w=10, v=0Same start
1w=6 (×0.6)v=0+20=20; w=10−0.2×20=6Same first step
2w=3.6 (×0.6)v=0.9×20+12=30; w=6−0.2×30=0Momentum much faster!
Normal GD: w goes 10→6→3.6. Momentum: w goes 10→6→0 (hits minimum in 2 steps!). Past gradients built up speed in the right direction.

Why Momentum Helps with Bad Conditioning

Without Momentum

GD zig-zags between the walls of the elliptical valley. Each step, it overshoots in the steep direction, then has to come back.

With Momentum

Oscillations in the steep direction cancel out (positive then negative gradients average to ~zero). Movement in the flat direction accumulates. Path becomes smooth!

💡 Typical values: β=0.9 (standard), β=0.95 or 0.99 (for very noisy gradients). In PyTorch: optim.SGD(params, lr=0.1, momentum=0.9)
🧠 Quick Check

What happens to the velocity v_t in the momentum formula if we set β = 0?

A The optimizer becomes Adam
B v_t = gradient alone → reduces to vanilla gradient descent
C The optimizer diverges
D Learning rate becomes 0
With β=0: v_t = 0·v_{t-1} + ∇J = ∇J. No memory of past gradients. The update becomes w = w − η·∇J, which is exactly vanilla gradient descent!
5.4 · Advanced Optimizers

Adagrad → RMSProp → Adadelta → Adam

The problem: one learning rate for all parameters is suboptimal. Some parameters need big updates (rare features/pixels), some need small updates (frequent ones). Adaptive methods give each parameter its own effective learning rate.

🗣️
Analogy — The Quiet Student in Class

In a class discussion, the student who talks constantly gets "tuned out" — people stop listening as closely. But the quiet student who rarely speaks — when they do speak, everyone pays attention! Adaptive optimizers do the same: parameters that "scream" (frequent large gradients) get their learning rate reduced. Quiet parameters get more attention when they do speak.

1. Adagrad — Adaptive Gradient

For each parameter i: s_{t,i} = s_{t-1,i} + g²_{t,i} ← running sum of squared gradients Update: θ_{t+1,i} = θ_{t,i} − (η / √(s_{t,i} + ε)) · g_{t,i} Intuition: Large past gradients → big s_{t,i} → small effective LR (frequent = slow down) Small past gradients → small s_{t,i} → big effective LR (rare = speed up)
🔢 Adagrad: f(w)=w², w₀=10, η=1
Stepwg=2ws = s_prev + g²Effective LR = η/√sw_new

LR automatically decays: 0.050 → 0.037 → 0.032. But notice s only grows! Eventually s→∞ and LR→0 — learning stops. This is Adagrad's main weakness.

2. RMSProp — Leaky Adagrad

Fix: instead of summing all squared gradients, use an Exponential Moving Average (EMA) — old gradients "leak out" and lose influence.

s_{t,i} = ρ·s_{t-1,i} + (1-ρ)·g²_{t,i} ← EMA (ρ ≈ 0.9) Update same as Adagrad: θ_{t+1,i} = θ_{t,i} − (η/√(s_{t,i}+ε)) · g_{t,i} Key difference from Adagrad: s_t doesn't keep growing forever! Old gradients fade with factor ρ per step → LR stays adaptive and never hits zero
🔢 RMSProp Step 1: w₀=10, η=0.1, ρ=0.9, s₀=0
g₁ = 2×10 = 20 s₁ = 0.9×0 + 0.1×20² = 0 + 40 = 40 (NOT 400 like Adagrad!) η_eff = 0.1/√40 ≈ 0.1/6.32 ≈ 0.0158 Δw = 0.0158 × 20 ≈ 0.316 w₁ = 10 − 0.316 = 9.684 (vs Adagrad's 9.0)
s₁=40 for RMSProp vs 400 for Adagrad (10× smaller!) because EMA doesn't blindly accumulate. This keeps LR from dying too fast.

3. Adadelta — No Global Learning Rate

Adadelta takes this further: no global η at all! The learning rate emerges automatically from the ratio of two EMAs:

E[g²]_t = ρ·E[g²]_{t-1} + (1-ρ)·g²_t ← EMA of squared gradients E[Δw²]_t = ρ·E[Δw²]_{t-1} + (1-ρ)·Δw²_t ← EMA of squared updates Effective step: Δw_t = − (√E[Δw²]_{t-1} + ε) / (√E[g²]_t + ε) · g_t Intuition: ratio of "how big updates have been" to "how big gradients are" No η to tune! But Adam has largely replaced it in practice.

4. Adam — The Industry Standard ⭐

Adam = Adaptive Moment Estimation. It combines: (1) Momentum (first moment = EMA of gradients) + (2) RMSProp (second moment = EMA of squared gradients) + (3) Bias correction.

Given gradient g_t at step t: (1) First moment (momentum): m_t = β₁·m_{t-1} + (1-β₁)·g_t [β₁ ≈ 0.9] (2) Second moment (RMSProp): v_t = β₂·v_{t-1} + (1-β₂)·g²_t [β₂ ≈ 0.999] (3) Bias correction (because m₀=v₀=0, early estimates are too small): m̂_t = m_t / (1 − β₁ᵗ) v̂_t = v_t / (1 − β₂ᵗ) (4) Update: w_{t+1} = w_t − η · m̂_t / (√v̂_t + ε) Typical values: η=1e-3, β₁=0.9, β₂=0.999, ε=1e-8
🔢 Adam Full Example: f(w)=w², w₀=1, η=0.1, β₁=0.9, β₂=0.999

Step 1 (t=1):

g₁ = 2×1 = 2 m₁ = 0.9×0 + 0.1×2 = 0.2 v₁ = 0.999×0 + 0.001×4 = 0.004 m̂₁ = 0.2/(1−0.9¹) = 0.2/0.1 = 2.0 ← bias corrected v̂₁ = 0.004/(1−0.999¹) = 0.004/0.001 = 4.0 Step = 0.1 × 2.0/√4.0 = 0.1 × 2/2 = 0.1 w₁ = 1 − 0.1 = 0.9

Step 2 (t=2):

g₂ = 2×0.9 = 1.8 m₂ = 0.9×0.2 + 0.1×1.8 = 0.18+0.18 = 0.36 v₂ = 0.999×0.004 + 0.001×3.24 ≈ 0.007236 m̂₂ = 0.36/(1−0.81) = 0.36/0.19 ≈ 1.895 v̂₂ = 0.007236/(1−0.998) ≈ 3.62 → √v̂₂ ≈ 1.903 Step = 0.1 × 1.895/1.903 ≈ 0.0996 w₂ = 0.9 − 0.0996 ≈ 0.800
Adam smoothly moves w: 1 → 0.9 → 0.80, heading toward minimum at 0. The bias correction prevented wildly large steps in the first few iterations!

  • DL Pipeline: Input → Network → Prediction → Loss → Optimizer → Repeat
  • GD Update: W_new = W_old − η · ∇L(W_old)
  • 3 Problems: Local minima, Saddle points, Vanishing gradients
  • Vanishing gradient fix: ReLU (derivative=1) + ResNet skip connections
  • Convex functions: Local min = Global min. f''≥0 for convex. Jensen: E[f(X)] ≥ f(E[X])
  • GD variants: Batch (stable, slow) → Mini-batch (best!) → SGD (fast, noisy)
  • LR Scheduling: Constant → Exponential → Cosine Annealing
  • Momentum: v_t = β·v_{t-1} + ∇J; w = w − η·v_t; helps zig-zagging
  • Adagrad: Per-param LR via squared gradient sum (LR → 0 eventually)
  • RMSProp: Adagrad with EMA — LR stays healthy
  • Adam: Momentum + RMSProp + Bias correction — the default optimizer

Week 6
Generative Models: Teaching AI to Create
6.1 · Foundations

Generative vs. Discriminative Models

This is the first big conceptual split in Week 6. Until now, we've built discriminative models (classify this!). Generative models learn something much richer — the distribution of the data itself.

🎨
Analogy — Art Critic vs. Artist

Discriminative model = Art critic: Given a painting, says "This is Monet" or "This is Picasso". Learns to distinguish.

Generative model = Artist: Has studied so many Monet paintings that it can paint a new Monet from scratch — one that never existed but looks authentic!

Discriminative — Learns P(Y|X)

  • Given X, what is the label Y?
  • Learns decision boundary
  • Can't generate new data
  • Examples: Logistic Regression, SVM, CNN classifiers

Generative — Learns P(X,Y) or P(X)

  • What does the data look like?
  • Learns full distribution
  • Can generate, inpaint, detect anomalies
  • Examples: GAN, VAE, Diffusion Models

Generative Models — What Can They Do?

1
Generate new realistic samples: Create images/audio/text that look like training data but aren't in it. (Stable Diffusion, DALL·E, Sora)
2
Inpainting (fill in gaps): Given a partial image, predict what the missing parts look like. Used in photo restoration!
3
Anomaly Detection: If the model has learned P(X) for "normal" data, an input with very low P(X) is an anomaly/outlier.
4
Representation Learning: Learn a meaningful latent space — a compressed representation where similar data is close together.

Latent Space — The "Code" of the World

🧬
Analogy — DNA as a Latent Space

A complete human (10 trillion cells, impossibly complex) is encoded in ~3 billion DNA base pairs. That's a compressed representation! Similarly, a 1024×1024 image (3M pixels) can be represented in a 512-dimensional latent vector z. Changing z slightly = changing facial expression, lighting, age, etc. in a smooth, meaningful way.

Generative Model Structure: Encoder: X (image) → z (latent vector, e.g. 512-D) Generator: z (latent vector) → X' (new image) z ~ N(0, I) ← sample random noise X' = Generator(z) ← decode to new image! The magic: z is in a "semantic space" where: - Moving along one dimension = changing smile intensity - Moving along another = changing hair color - Interpolating between two z vectors = morphing between faces
🧠 Quick Check

What is the fundamental difference between discriminative and generative models?

A Discriminative models are newer and more powerful than generative models
B Generative models can only work with images, not text
C Discriminative models learn P(Y|X) while generative models learn P(X) or P(X,Y)
D Generative models require labeled data; discriminative models do not
Discriminative models learn the conditional probability of labels given data P(Y|X) — they just learn to classify. Generative models learn the data distribution P(X) or joint P(X,Y), allowing them to generate new data, fill gaps, detect anomalies, and more!
6.1 · Math of Generation

Probability, Marginalization & Inpainting

Let's build intuition through a tiny toy example: a 2×2 binary image (4 pixels, each 0 or 1). This gives us only 2⁴ = 16 possible images — small enough to reason about completely.

The Toy World: 2×2 Binary Image

Image: 4 pixels → x₁₁, x₁₂, x₂₁, x₂₂ (each is 0 or 1) Example image: [1 1] [0 0] → stored as vector [1, 1, 0, 0] Total possible images = 2⁴ = 16 A Generative Model assigns a probability to EACH of the 16 patterns! (Thinks: "Which patterns are common in the real world?")
📊 Joint Distribution P(x₁₁, x₁₂, x₂₁, x₂₂) — 8 of 16 patterns shown
Pattern (x₁₁x₁₂x₂₁x₂₂)Image looks likeP(pattern)
1 0 0 0Top-left only lit0.05

Marginalization — Finding the Probability of One Pixel

Question: What's the probability that the top-left pixel x₁₁ = 1? We sum over all possible values of the other pixels:

P(x₁₁=1) = Σ over all x₁₂,x₂₁,x₂₂ of P(x₁₁=1, x₁₂, x₂₁, x₂₂) = 0.05 + 0.08 + 0.10 + 0.04 + 0.12 + 0.06 + 0.09 + 0.11 = 0.65 Interpretation: In this toy world, 65% of images have the top-left pixel lit.
📊
Analogy — "What's the chance it rains today?"

You don't know the temperature, wind speed, or humidity — but you can still ask "what's the chance of rain?" by summing over all possible combinations of those other variables. That's exactly marginalization — sum (or integrate) over the variables you don't care about to get the probability of what you do care about.

Inpainting — Filling in Missing Pixels

Now the key application: given that x₁₁=1 (top-left pixel is white), what's the most likely completion of the image?

We want: argmax P(x₁₂, x₂₁, x₂₂ | x₁₁=1) Using Bayes' theorem: P(x_missing | x_observed) = P(x_missing, x_observed) / P(x_observed) Since P(x₁₁=1) = 0.65 is constant, we just need to find which pattern with x₁₁=1 has the highest joint probability: Pattern | P 1 0 0 0 | 0.05 1 0 0 1 | 0.08 1 0 1 0 | 0.10 1 0 1 1 | 0.04 1 1 0 0 | 0.12 ← MAXIMUM! 🏆 1 1 0 1 | 0.06 1 1 1 0 | 0.09 1 1 1 1 | 0.11 Answer: Best completion = [1 1; 0 0] (top row all white, bottom all black)
🔢 Generative Classifier — Bayes Theorem Applied

A generative model learns P(X|class) for each class. To classify, use Bayes' theorem:

Given: P(X|cat)=0.02, P(X|dog)=0.01, P(cat)=0.6, P(dog)=0.4 Score_cat = P(X|cat) × P(cat) = 0.02 × 0.6 = 0.012 Score_dog = P(X|dog) × P(dog) = 0.01 × 0.4 = 0.004 Since 0.012 > 0.004 → Predict: CAT 🐱 Exact posterior: P(cat|X) = 0.012 / (0.012 + 0.004) = 0.012/0.016 = 75% P(dog|X) = 25%
This is how classic Naive Bayes classifiers work! A generative model that models P(X|Y) can be used for classification via Bayes' theorem.

The Scaling Problem — Why We Need Neural Networks

⚠️ The Curse of Dimensionality: Our toy 2×2 binary image had only 2⁴ = 16 states — easy! But a real 32×32 grayscale image has 256^(32×32) ≈ 10^2466 possible images. We can NEVER store this table. This is why we use neural networks that approximate the distribution function instead of storing it explicitly.
GAN
Generative Adversarial Network: Generator creates fake images, Discriminator tries to spot them. They compete until the generator makes perfect fakes. (Noise z → Generator → Fake image)
VAE
Variational Autoencoder: Encoder compresses image to latent z, Decoder reconstructs. z is a smooth, structured latent space you can interpolate through.
Diff
Diffusion Models: Gradually add noise to an image (destroys it), then train a network to reverse the process (denoise). Stable Diffusion, DALL·E 3, Midjourney all use this!
🧠 Quick Check

In the inpainting problem, given observed pixels, how do we find the best completion?

A Choose random values for missing pixels
B Always use zeros for missing pixels
C Minimize P(missing pixels | observed pixels)
D Find the missing values that maximize P(missing|observed), equivalent to maximizing the joint probability
We want the most likely completion: argmax P(x_miss | x_obs). By Bayes' theorem, this equals argmax P(x_miss, x_obs) / P(x_obs). Since P(x_obs) is constant, we just find the pattern with the highest joint probability among all completions of the known pixels!
📋 Week 6 Summary
  • Discriminative: Learns P(Y|X) — classifies, can't generate. (CNNs, logistic regression)
  • Generative: Learns P(X) or P(X,Y) — can generate, inpaint, detect anomalies, classify via Bayes
  • Latent space z: Compressed, meaningful representation. z ~ N(0,I) → Generator → new image!
  • Marginalization: P(x₁₁=1) = sum over all other pixel combos of P(joint)
  • Inpainting: argmax P(missing | observed) = argmax joint probability among valid completions
  • Generative classifier: Score_c = P(X|c) × P(c) → predict class with highest score (Bayes' theorem)
  • Scaling problem: Real images have 2^(millions) states → use neural nets to approximate P(X), not store it
  • Modern generative models: GAN (adversarial), VAE (variational autoencoder), Diffusion (Stable Diffusion, DALL·E 3)

Adam (Adaptive Moment Estimation) = Momentum (first moment/EMA of gradients) + RMSProp (second moment/EMA of squared gradients) + bias correction for both. It's the default optimizer for a reason!

2. A loss function L(w) = w⁴ + w² has gradient 4w³ + 2w = 0 only at w=0. Is w=0 a local minimum, saddle point, or global minimum?

A Saddle point — it looks like a minimum but isn't
B Global minimum — the function is always ≥ 0 and equals 0 only at w=0
C Local minimum but not global minimum
D Not a critical point at all
L(w) = w⁴ + w² ≥ 0 for all w, and equals exactly 0 only when w=0. So w=0 is the global minimum! Also, L''(w) = 12w²+2, and L''(0) = 2 > 0, confirming it's a minimum not a saddle point.

3. In the 2×2 binary image example, P(x₁₁=1) was calculated as 0.65. What mathematical operation was used?

A Conditional probability using Bayes' theorem
B Maximum likelihood estimation
C Chain rule of probability
D Marginalization — summing the joint probability over all values of the other variables
Marginalization! P(x₁₁=1) = Σ over all (x₁₂,x₂₁,x₂₂) of P(x₁₁=1, x₁₂, x₂₁, x₂₂). We summed all 8 joint probability entries where x₁₁=1 to get 0.65. This is a fundamental operation in probability theory!

4. Why can't we store the complete probability table for a 32×32 grayscale image?

A 32×32 image has 1024 pixels, each taking 256 values → 256^1024 ≈ 10^2466 possible images. That's incomprehensibly larger than the estimated 10^80 atoms in the observable universe! This is why generative models must learn a function that approximates P(X), not a lookup table.
